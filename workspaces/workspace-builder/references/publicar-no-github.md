# Publicar o seu workspace no GitHub (bloco opcional)

Guia dos 20 minutos finais da oficina, para quem quiser levar o próprio workspace para casa versionado. É opcional. Quem não fizer agora pode fazer depois.

## O que é isso, em uma frase

O GitHub é um lugar na internet onde o seu conjunto de pastas fica guardado com histórico. Cada vez que você salva uma mudança (um "commit"), ele grava quem fez o quê e quando. Você ganha rastreabilidade: consegue voltar no tempo e ver como o trabalho evoluiu.

## Passo a passo

1. **Ter uma conta.** Se ainda não tem, crie uma conta gratuita em github.com. Se você usou o Copilot na oficina, provavelmente já entrou com uma conta GitHub.
2. **Entrar no VS Code.** No canto inferior esquerdo do VS Code, clique no ícone de conta e faça login no GitHub, se ainda não estiver conectado.
3. **Abrir o controle de versão.** Clique no ícone de ramificação (Source Control) na barra lateral esquerda.
4. **Publicar.** Clique em "Publish to GitHub". Escolha um nome para o repositório e se ele será público ou privado (privado é uma boa escolha para começar).
5. **Pronto.** O VS Code envia o seu workspace para o GitHub. A partir daí, cada mudança que você salvar pode ser enviada com um clique.

## Por que vale a pena

- O produto já é seu: você leva a própria célula embora.
- Toda mudança fica registrada em uma fila que você pode rastrear ao longo do tempo.
- Você pode abrir o mesmo workspace em outra máquina, ou compartilhá-lo com um colega.

## Se der algum erro

Chame um facilitador. Autenticação de GitHub costuma ser o ponto que mais trava, e resolvemos rápido junto com você.
